Регистрация:
1. Пользователь открывает https://localhost:3000/sign_up
2. Загружается HTML страница с полями (Email, Name, Password, Password reply)
3. После заполнения всех обязательных полей пользователь нажимает кнопку отправить.
4. Проверяется есть ли данный Email уже в БД, соответствует ли длинна пароля нужной мин 6 символов.
4.1. Если данный email уже используется тогда рендерим страницу регистрации пользователю опять с указанием ошибки.
4.2. Если все формы заполнены правильно тогда хешируем пароль и повтор пароля.
4.3. Если хешируемый пароль и хешируемый повтор пароля совпадают переходим к следующему пункту.
5. Если все прошло хорошо возвращаем пользователю https://localhost:3000 и сообщаем о его успешной регистрации.

Вход:
1. После нажатия на "Sign in" отправляется запрос на сервер https://localhost:3000/sign_in
2. Сервер возвращет HTML форму входа в систему.
3. Пользователь заполняет поля "email" и "password" и отправлет запрос на сервер.
4. Сервер находит пользователя по его указанному email.
5. Сервер хеширует пароль который прислал пользователь.
6. Сравниваем пароль из БД и тот который прислал пользователь.
7. Если к указанному емейл адресу пароли не совпадают возвращяем сообщение об ошибке.
8. Если Все данные верны, пользователь успешно входит в систему.

Обычный пользователь:

1. Пользователь открывает https://localhost:3000
2. Товар подгружается с БД.
3. Товар отображается в HTML блоке.
4. Страница HTML грузится и пользователь видет весь товар.

Открытие товара:

1. Пользователь нажимает на любой товар.
2. Отправляется на сервер запрос https://localhost:3000/products/id###
3. Сервер находит в БД нужный товар с его картинкой и остальными данными о товаре.
4. Вставляется это все в HTML шаблон.
5. Шаблон который вернул сервер рендерится на стороне пользователя.

Действия пользователя вошедшого в систему:

1. На главной localhost:3000 и в каждом товаре /products/id### видет кнопку "Add to favorites" и "My favorites"
2. При нажатии на "Add to favorites" запрос обрабатывается сервером.
3. И добавляет товар в избранное именно этого пользователя.
4. При посещении "My favorites", сервер достает из базы все товары которые пользователь добавил в избранное.
5. Когда пользователь убирает товар из избранного, сервер удаляет из колонки "favorites" id данного товара по id пользователя.

Админ удаление:

1. На каждом товаре есть кнопка "Delete".
2. При ее нажатии всплывает окно об потверждении удаления.
3. Сервер обрабатывает запрос на метод destroy данного ID товара.
4. При нажатии на "ОК" данный товар с данным ID удаляется из базы.

Админ добавление:

1. При добавления товара https://localhost:3000/new
2. Сервер обрабатывает запрос "new" возвращяет форму для нового товара.
3. Заполнение всех полей и добавления картинки (name, price, img)
4. Если какое-то поле не введено сервер возвращяет ответ с ошибкой что не было сделано.
5. Если все поля были введены корректно данная запись сохраняется в базе данным.
